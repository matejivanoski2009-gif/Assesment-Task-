# Software Development - One Page Report:
## Purpose, Design Decisions, and Functionality:

## Purpose:
The purpose of 'Algorithm Academy' is to aid Software Engineering Students in better understanding algorithms more engagingly. Lots of students find concepts such as sequence, selection, iteration, subprograms and others confusing and complicated when they are shown as written pseudocode or chunks of code. The objective of this system is to make those ideas clearer.

Students actively participate in the learning process. For instance, they may be asked to arrange the steps of an algorithm into the correct order, or predict what a loop will produce. By making decisions and receiving feedback immediately, students reinforce their understanding of how algorithms work. Users will answer multiple-choice questions and receive feedback after every response. The addition of rewards such as badges and XP encourages students to continue practising and improving over time. 

## Design Decisions:
The system will be developed as a web application so it can run in any web browser. There are a variety of different coding languages that will be implemented, such as HTML, JavaScript and CSS. JavaScript will act as the "brains," implementing the core game logic and enabling interactive elements to respond to user actions. HTML will be utilised to structure the information within the website. CSS will be implemented to design the appearance of the web application, and JavaScript is used to control the game logic, question progression, timers, login system and more.

In addition, the format of a game/quiz was chosen due to the fact that it makes a better learning experience for Software Engineering students. When learning feels interactive and fun, students have a higher chance of achieving the maximum they can. 

Several decisions within the interface were also made to improve User Experience (UX). Dark mode was chosen to allow users more control over the application. Login and sign-up pages were included so that users can save their XP and badges, making the system feel more personalised. The overall design will focus on simplicity and usability so that students can focus on learning rather than going through a complicated website.

## Functionality:
Furthermore, the user will interact with the website by selecting, clicking and entering. Users will interact with 'Algorithm Academy' by entering a username, selecting a topic and answering multiple choice questions. he system will check the response via the conditional statements and immediately provide feedback demonstrating whether the response is correct or incorrect. 

The system will keep track of their score while the student plays. This will be achieved through Iteration to move through multiple challenges in sequence. After all the questions are done, the system will display a result illustrating the score on a leaderboard compared to other users and a chance to retry any incorrect questions. There will also be an option to start again. Throughout the quiz, the web application awards XP and an opportunity to get badges. The profile page stores and displays the user's current XP, level rank, high score and badges.
