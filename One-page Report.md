# Software Development - One Page Report:
## Purpose, Design Decisions, and Functionality:

## Purpose:
The purpose of this interactive website is to aid Software Engineering Students in better understand algorithims more engagingly. Lots of students find algorithms confusing and complicated when they are shown as written pseudocode or chunks of code. Ideas like loops, variables, events, arrays/lists, conditional statements and more can be difficult to visualise. The objective of this system is to make those ideas clearer.

Students actively participate in the learning process. For instance, they may be asked to arrange the steps of an algorithm into the correct order, or predict what a loop will output. By making decisions and receiving feedback immediately, students reinforce their understanding of how algorithms work. 

## Design Decisions:
The system will be developed as a web application so it can run in any web browser. There are a variety of different coding languages that will be implemented, such as HTML, JavaScript, CSS and Python. JavaScript will act as the "brains," implementing the core game logic and enabling interactive elements to respond to user actions. HTML will be utilised to structure the information within the website, and CSS will be utilised to build a clear and well-structured format.

In addition, the format of a game/quiz was chosen due to the fact that it makes a better learning experience for Software Engineering students. When learning feels interactive and fun, students have a higher chance of achieving the maximum they can. Subprograms will be used to organise different parts of the system, like checking answers and updating the score. 

The overall design will focus on simplicity and usability so that students can focus on learning rather than going through a complicated website.

## Functionality:
Furthermore, the user will interact with the website by selecting, clicking and/or entering. The system will check the response via the conditional statements and immediately provide feedback demonstrating whether the response is correct or incorrect.

The system will keep track of their score while the student plays. This will be achieved through Iteration to move through multiple challenges in sequence. After all the challenges are done, the system will display a result illustrating the score and underneath an option to retry or quit.
