# Software Development:
## Requirements Definition:
### Purpose and Objectives:
'Algorithm Academy' will be an interactive educational website designed for Year 11 Software Engineering Students about algorithms. The purpose of the system is to aid students in understanding key ideas like sequence, iterations, selection and other terms that enable algorithms to run via short interactive quizzes. The objective of this system is to transform those concepts into a more engaging learning experience via quizzes, instant feedback and rewards. The web application aims to increase motivation, improve knowledge and create a fun environment for students.

### Problem Statement:
Software Engineering students find algorithm topics challenging as they can find the topic too difficult to understand and not interactive enough. As a result, students struggle to apply these concepts to real-life scenarios such as exams and assessments. There is a need for an educational tool that allows students to actively test themselves and expand their knowledge. Algorithm Academy solves this by combining work with a quiz game that is both educational and engaging.

Functional Requirements involve:
- Must allow users to create an account, sign up and log in
- Must allow users to select quiz topics
- Must display multiple-choice questions
- Must include a timer for each question

Non-functional requirements involve:
- Must be simple and easy to use
- Must include readable fonts, clear buttons and clear colours
- Must run smoothly
- Code should remain organised

### Acceptance Criteria:
The project will be considered successful if users can create an account, log in and begin the quiz.
The system must correctly move users through quiz questions.
XP, steaks, badges and other profile data must update correctly after each attempt of the quiz.
Testing must show that the website performs smoothly without any errors.

### Glossary:

- Algorithm --> A set of steps used to solve a problem.
- Sequence --> Instructions performed in order.
- Selection --> A decision made in a program.
- Iteration --> Repetition of instructions through loops.
- Subprograms --> A smaller section of code designed to perform a task.
- Validation --> Checking whether the data entered is correct.
